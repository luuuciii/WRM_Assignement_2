Git / Collaboration Workflow
The lecturer already describes the exact workflow you should follow. The cleanest split for two people:

Person	Owns
You (A)	Tasks 1, 2, 3
Friend (B)	Tasks 4, 5
Branch naming: one branch per task section (e.g. task1-review, task2-modelling, etc.), each opened as a Pull Request, reviewed by the other person, then merged to main. This satisfies the requirement that each person branches and merges at least once, while keeping main always runnable.

Critical note: .py files are git-friendly (plain text diffs); the .ipynb is not. Keep all logic in the .py helpers. The notebook only calls functions and displays results — this prevents merge conflicts.

Task-by-Task Outline
Task 1 — Timeseries Review [2 pts] → task1_review.py
Load & resample all 4 CSVs (Q+C for both stations) to monthly means
Handle gaps: Gisingen-C has a gap in 2019, Gisingen-Q ends 2013; Diepoldsau-C starts 2012. Monthly resampling with min_count=1 handles sparse months cleanly.
Plot all 4 timeseries full length
Linear regression on each series (OLS slope + t-test at 5% significance)
Detrend: subtract mean (if no sig. trend) or the fitted trend line
Discuss: is the Q trend at Gisingen (1976–2013) physical (glacier retreat, land use) or instrumental?
Key output: 4 detrended, zero-mean timeseries with their estimated variance σ²

Task 2 — Timeseries Modelling [5 pts] → task2_modelling.py
ACF + PACF for all 4 series, with 95% CI bands (±1.96/√N)
Gap handling per lecturer's note: for Gisingen, compute ACF/PACF separately before and after the gap years, then average
Select AR order p: where PACF cuts off
Select ARMA order (p,q): where ACF tails off + PACF tails off — use AIC/BIC to compare
Fit AR and ARMA using statsmodels.tsa.arima.model.ARIMA
Discuss: can the same order be used for Q at both stations? Same for C?
Seasonal note: monthly data is seasonal — either fit on anomalies (subtract monthly climatology) or accept order ≤ 12 as instructed
Task 3 — Application & Evaluation [5 pts] → task3_evaluation.py
Plot empirical ACF vs. theoretical ACF of each fitted model on one figure (4 series × 2 models = 8 plots, or 4 subplots with both models overlaid)
Plot residual ACF for each fitted model with 95% CI
Ljung-Box (Portmanteau) test at 5% level for residual independence
Probability plot (Q-Q plot) + PPCC test for residual normality
Discussion: choose best model (AR or ARMA) per variable; check parsimony; adjust order if warranted
Task 4 — Ill to Rhein Sediment Influence [5 pts] → task4_simulation.py
Generate 10 synthetic timeseries of 10 years (120 months) each from the chosen model, for each of the 4 series
Plot synthetic series alongside normalized historical data (zero-mean, no trend added back)
Compare statistics: mean, std, ACF shape
Compute sediment mass M(t) = C(t) · Q(t) [kg/s] — requires overlapping C and Q period
Monthly and yearly average sediment yield from synthetic series, compared to observed
Ill contribution to Diepoldsau: ratio of Ill sediment mass to Rhein sediment mass
Discuss non-synchronous series implications
Note: For Gisingen, Q is available 1976–2013 and C is 2003–2021 — overlap is 2003–2013. Use this window for M(t).

Task 5 — Independent Variables? [3 pts] → task5_independence.py
Pearson and Spearman correlation between Q and C at each station (using overlapping period)
Joint distribution scatter plot (Q vs. C) at each station
Independence test: Chi-squared or mutual information; or simply show correlation ≠ 0 is strong quantitative evidence
Discuss: simulating Q and C with independent AR/ARMA models ignores their positive correlation → underestimates sediment peaks; resolution = bivariate/copula model or VAR model
